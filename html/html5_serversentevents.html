Для чого потрібні Server-Sent Events (SSE) - Події, надіслані сервером?

Події, надіслані сервером дозволяють веб-сторінці отримувати оновлення з сервера.

Події, надіслані сервером - це односторонні повідомлення

Подія, надіслана сервером - це коли веб-сторінка автоматично отримує оновлення з сервера.

Це також було можливим і раніше, але на веб-сторінці потрібно було запитати, чи доступні оновлення. З подіями, надісланими сервером, оновлення надходять автоматично.

Приклади: Facebook/Twitter оновлення, оновлення цін на акції, стрічки новин, спортивні результати тощо.

Отримувати сповіщення про надіслані сервером події

Об'єкт EventSource використовується для отримання повідомлень про події, які надсилаються сервером:

Приклад:

var source = new EventSource("demo_sse.php");
source.onmessage = function(event) {
  document.getElementById("result").innerHTML += event.data + "<br>";
};

Пояснення прикладу:

  • Створіть новий об'єкт EventSource і вкажіть URL-адресу сторінки, що надсилає оновлення (у цьому прикладі "demo_sse.php")
  • Кожного разу при отриманні оновлення відбувається подія onmessage
  • Коли відбувається подія onmessage, помістіть отримані дані до елемента з id = "result"

Приклад коду на стороні сервера

Для наведеного вище прикладу потрібен сервер, який може надсилати оновлення даних (наприклад, PHP або ASP).

Синтаксис потоку подій на стороні сервера простий. Встановіть заголовок "Content-Type" на "text/event-stream". Тепер ви можете почати відправляти потоки подій.

Код на PHP (demo_sse.php):

<?php
header('Content-Type: text/event-stream');
header('Cache-Control: no-cache');

$time = date('r');
echo "data: The server time is: {$time}\n\n";
flush();
?>

Код на ASP (VB) (demo_sse.asp):

<%
Response.ContentType = "text/event-stream"
Response.Expires = -1
Response.Write("data: The server time is: " & now())
Response.Flush()
%>

Пояснення коду:

  • Встановіть заголовок "Content-Type" в "text/event-stream"
  • Вкажіть, що сторінка не повинна кешуватись (not cache)
  • Виведіть дані, які потрібно надіслати (Завжди починати з "data: ")
  • Скиньте вихідні дані назад на веб-сторінку

Об'єкт EventSource

У наведених вище прикладах ми використовували подію onmessage для отримання повідомлень. Але доступні й інші події:

Події Опис
onopen Коли відкрито з'єднання з сервером
onmessage Коли отримано повідомлення
onerror Коли виникла помилка
