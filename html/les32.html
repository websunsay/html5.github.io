HTML/CSS. URL або веб-адреса

URL-адреса - це інша назва для веб-адреси. URL-адреса може складатися зі слів (w3schools.com) або адреси Інтернет-протоколу (IP-адреса) (192.68.20.50). Більшість людей вводять ім'я під час серфінгу, тому що імена легше запам'ятовувати, ніж числа.

URL - уніфікований локатор ресурсів

Веб-браузери запитують сторінки з веб-серверів за допомогою URL-адреси. Уніфікований інформаційний ресурс (URL) використовується для адреси документа (або інших даних) в Інтернеті. Веб-адреса, наприклад, https://www.w3schools.com/html/default.asp, дотримується цих правил:

scheme://prefix.domain:port/path/filename

Пояснення:

  • scheme (схема) - визначає тип Інтернет-служби (найбільш поширеним є http або https)
  • prefix (префікс) - визначає префікс домену (за умовчанням для http є www)
  • domain (домен) - визначає назву домену Інтернету (наприклад, w3schools.com)
  • port (порт) - визначає номер порту на хості (за замовчуванням для http - 80)
  • path (шлях) - визначає шлях на сервері (якщо пропущено: кореневий каталог сайту)
  • filename (назва файлу) - визначає назву документа або ресурсу

Загальні схеми URL-адрес

У таблиці нижче наведено деякі загальні схеми:

Схема Коротко для Використовується для
http Протокол передачі гіпертексту Загальні веб-сторінки. Не зашифровано
https Безпечний протокол передачі гіпертексту Захищає веб-сторінки. Зашифровано

URL кодування

URL-адреси можна надсилати лише через Інтернет за допомогою набору символів ASCII. Якщо URL-адреса містить символи поза набором ASCII, URL-адресу потрібно конвертувати.

Кодування URL перетворює не-ASCII символи у формат, який може бути переданий через Інтернет.

Кодування URL замінює символи не-ASCII на "%", за якими йдуть шістнадцяткові цифри.

URL-адреси не можуть містити пробіли. Кодування URL зазвичай замінює пробіл знаком плюс (+) або %20.

Спробуйте самі

Якщо натиснути кнопку "Відправити", браузер буде кодувати URL-адресу у вхідний сигнал, перш ніж він буде надісланий на сервер. Сторінка на сервері відобразить отриманий вхід.

Спробуйте інший вхід і ще раз натисніть "Відправити".
