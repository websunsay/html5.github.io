HTML Резюме

Цей підручник навчив вас використовувати HTML для створення власного веб-сайту.

HTML є універсальною мовою розмітки для Web. HTML дозволяє форматувати текст, додавати графіку, створювати посилання, форми введення, фрейми та таблиці тощо, а також зберігати їх у текстовому файлі, який може читати та відображати будь-який веб-браузер.

Для отримання додаткової інформації про HTML, будь ласка, зверніться до HTML прикладів та HTML довідника.

Тепер ви знаєте HTML, що далі?

Що необхідно вивчати після вивчення HTML?

Як правило, після вивчення HTML, вивчають також такі технології, як CSS та JavaScript (для фронтенд-розробників).

Вивчення CSS

CSS дозволяє стилізувати HTML-сторінки.

CSS надає повний контроль над макетом, не змінюючи змісту документа.

Щоб дізнатися більше про CSS, відвідайте CSS підручник.

Вивчення JavaScript

JavaScript робить сайт більш динамічним. Динамічний веб-сайт може реагувати на події та дозволяти взаємодію з користувачем.

JavaScript є найпопулярнішою мовою сценаріїв в Інтернеті, і він працює з усіма основними браузерами.

Якщо ви хочете дізнатися більше про JavaScript, будь ласка, відвідайте JavaScript підручник.

Публікація веб-сайту

Щоб зробити свій веб-сайт доступним для кожного в світі, його потрібно опублікувати в мережі.

Для цього у вас є два варіанти:

  • Використовуйте постачальника послуг Інтернету (сервіс-провайдера);
  • Власний хостинг веб-сайту.

Використання послуг Інтернет сервіс-провайдера

Інтернет-провайдер (ISP) - це компанія, яка надає послуги доступу до Інтернету та використання Інтернету.

Інтернет-послуги, які зазвичай надаються Інтернет-провайдерами, включають доступ до Інтернету, Інтернет-транзит, реєстрацію доменного імені, веб-хостинг, послугу Usenet і колокацію.

Використання постачальника послуг Інтернету (ISP - Інтернет сервіс-провайдера) є найпоширенішим варіантом.

Переваги:

  • Швидкість з'єднання - Провайдери мають дуже швидке з'єднання з Інтернетом;
  • Потужне обладнання - Провайдери мають потужні веб-сервери, які можуть спільно використовувати декілька клієнтів. Можна також очікувати ефективного балансування навантаження та необхідних серверів резервного копіювання;
  • Безпека та стабільність - Провайдери - це фахівці з веб-хостингу. Очікуйте більше 99% часу, найновіші виправлення програмного забезпечення та найкращий захист від вірусів.

Що треба враховувати?

  • 24-годинна підтримка - Провайдер повинен запропонувати 24-годинну підтримку. Безкоштовний телефон також може бути життєво важливим;
  • Щоденний бекап (резервне копіювання) - Провайдер повинен виконувати щоденну процедуру резервного копіювання;
  • Обсяг трафіку - Перевірте обмеження обсягу трафіку постачальника послуг Інтернету (не закінчуйте платити гроші за несподіваний високий трафік);
  • Обмеження пропускної здатності або вмісту - Перевірте пропускну здатність провайдера та обмеження вмісту (чи можна публікувати фотографії, відео чи звук?);
  • Можливості електронної пошти - Переконайтеся, що провайдер підтримує потрібні можливості електронної пошти;
  • Доступ до бази даних - Переконайтеся, що провайдер підтримує потрібний доступ до бази даних.

Хостинг власного сайту

Хостинг власного веб-сайту на вашому власному сервері також є можливим.

Що треба враховувати?

  • Витрати на обладнання - Щоб запустити "реальний" веб-сайт, ви повинні придбати потужне серверне обладнання (низьковартісний ПК не виконуватиме необхідну роботу). Вам також знадобиться постійне (24/7) високошвидкісне Інтернет-з'єднання;
  • Витрати на програмне забезпечення - Серверні ліцензії часто перевищують вартість клієнтських ліцензій. Серверні ліцензії також можуть мати обмеження на кількість користувачів;
  • Витрати на оплату праці - Не очікуйте низьких трудових витрат. Ви повинні встановити власне апаратне та програмне забезпечення. Ви також повинні мати справу з помилками і вірусами, і тримати ваш сервер постійно працюючим.
