HTML Доступність

Пишіть HTML з урахуванням доступності. Надайте користувачу гарний спосіб наваігації та взаємодії з вашим сайтом. Зробіть ваш HTML код якомога більш семантичним, щоб його було легко зрозуміти як відвідувачам, так і читачам екрану (скринрідерам).

Семантичний HTML

Семантичний HTML означає максимально можливе використання правильних елементів HTML для їх правильного призначення. Семантичні елементи є елементами зі значенням; якщо вам потрібна кнопка, використовуйте елемент <button> (а не <div>).

Семантичний

<button>Click Me</button>

Не семантичний

<div>Click Me</div>

Семантичний HTML дає контекст для скринрідерів, які читають вміст веб-сторінки вголос.

З прикладом кнопки на увазі:

  • За замовчуванням кнопки мають більш відповідний стиль
  • Скринрідер ідентифікує його як кнопку
  • Фокусовані
  • Клікабельні

Кнопка також доступна для людей, що покладаються на навігацію за допомогою клавіатури; на неї можна натискати як мишою, так і клавішами на клавіатурі, а також можна вводити вкладки між ними (за допомогою клавіші табуляції на клавіатурі).

Приклади не семантичних елементів: <div> та <span> - нічого не розповідають про свій зміст.

Приклади семантичних елементів: <form>, <table> та <article> - чітко визначають свій зміст.

Заголовки важливі

Заголовки визначаються за допомогою тегів від <h1> до <h6>:

Приклад:

<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6</h6>
Спробуйте самі »

Пошукові системи використовують заголовки для індексування структури та вмісту ваших веб-сторінок.

Користувачі переглядають ваші сторінки за заголовками. Важливо використовувати заголовки, щоб показати структуру документів та зв'язок між різними розділами.

<h1> заголовки слід використовувати для основних заголовків, після чого слід використовувати <h2> заголовки, та менш важливі <h3> і так далі.

Примітка: Використовуйте заголовки HTML лише для заголовків. Не використовуйте заголовки просто для створення тексту ВЕЛИКИМ або жирним.

Альтернативний текст

Атрибут alt надає альтернативний текст для зображення, якщо користувач з якоїсь причини не може його переглянути (через повільне з'єднання, помилку в атрибуті src або якщо користувач використовує скринрідер).

Значення атрибута alt має описувати зображення:

Приклад:

<img src="img_chania.jpg" alt="Flowers in Chania">

Якщо браузер не може знайти зображення, він покаже значення атрибуту alt:

Приклад:

Оголошення мови

Оголошення мови є важливим для скринрідерів та пошукових систем, і оголошується атрибутом lang. Використовуйте наведений нижче код для показу веб-сторінки англійською мовою:

<!DOCTYPE html>
<html lang="en">
<body>

...

</body>
</html>

Використовуйте чисту мову

Використовуйте чисту мову, яку легко зрозуміти, і намагайтеся уникати символів, які неможливо прочитати чітко за допомогою програми для читання з екрана (скринрідером). Наприклад:

  • Зробіть речення якомога коротшими.
  • Уникайте тире. Замість того, щоб писати 1-3, запишіть так: від 1 до 3
  • Уникайте скорочень. Замість того, щоб писати Feb, пишіть February
  • Уникайте сленгових слів

Назви посилань

Приклад:
